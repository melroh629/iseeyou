# 반응형 디자인 가이드 (Responsive Design Guide)

이 프로젝트는 **모바일 우선(Mobile-First)** 반응형 디자인 전략을 따릅니다.

## 뷰포트 제약 사항 (Viewport Constraints)

작은 폴더블 기기부터 대형 데스크탑 모니터까지 레이아웃 안정성을 보장하기 위해 특정 최소 및 최대 너비 제약을 적용합니다.

### 최소 너비 (Minimum Width): `320px`
- **대상 기기**: 지원하는 가장 작은 모바일 기기 (예: iPhone SE 1세대, 갤럭시 폴드 커버 화면).
- **구현 위치**: `app/globals.css`의 `body`에 `min-width: 320px` 적용.
- **동작 방식**: 320px 미만에서는 레이아웃이 깨지는 대신 가로 스크롤바가 표시됩니다.

### 최대 너비 (Maximum Width): `1920px`
- **대상 기기**: 표준 Full HD 데스크탑 모니터.
- **구현 위치**: `app/layout.tsx`에 `max-w-[1920px]` 래퍼(wrapper) 적용.
- **동작 방식**:
    - 콘텐츠는 중앙 정렬됩니다 (`mx-auto`).
    - 1920px보다 넓은 화면에서는 좌우에 여백이 나타납니다.

## 브레이크포인트 (Breakpoints - Tailwind CSS 기본값)

반응형 조정을 위해 표준 Tailwind CSS 브레이크포인트를 사용합니다:

| 브레이크포인트 | 너비 | 대상 기기 |
|:---|:---|:---|
| `sm` | 640px | 대형 폰 / 소형 태블릿 |
| `md` | 768px | 태블릿 (아이패드 세로) |
| `lg` | 1024px | 태블릿 (아이패드 가로) / 소형 노트북 |
| `xl` | 1280px | 노트북 / 데스크탑 |
| `2xl` | 1536px | 대형 데스크탑 |

## 모범 사례 (Best Practices)

1.  **모바일 우선 (Mobile First)**: 기본 모바일 뷰(320px+)에 맞춰 스타일링을 시작하고, 더 큰 화면은 `sm:`, `md:` 수정자를 사용하여 덮어씌웁니다.
2.  **유연한 단위 (Flexible Units)**: 제약 조건 내에서 적응할 수 있도록 내부 레이아웃에는 `%`, `flex`, `grid`를 사용합니다.
3.  **테스트 (Testing)**: 항상 **320px** (최소) 및 **1920px** (최대)에서 레이아웃을 확인하세요.
